# ADR-003: Armazenamento Local de Uploads com Multer

## Contexto

O sistema permite upload de arquivos para diversas entidades (ex: KYC, documentos, fotos). É necessário armazenar esses arquivos e manter metadados no banco.

## Decisão

Utilizar o middleware Multer para upload de arquivos, armazenando-os localmente em diretórios sob `public/uploads` ou `upload/` conforme o tipo. Metadados são salvos na coleção Upload. (Evidência: backend/middlewares/uploadMiddleware/, backend/models/coreModels/Upload.js)

## Alternativas Consideradas

- Armazenamento em nuvem (AWS S3, Azure Blob)
- Banco de dados para arquivos (GridFS)

## Trade-offs

- Armazenamento local é simples e rápido para desenvolvimento e pequenas implantações.
- Pode gerar problemas de escalabilidade e backup em produção.
- Armazenamento em nuvem oferece alta disponibilidade e escalabilidade, porém com custo e complexidade.

## Consequências

- Necessidade de configurar backups e monitorar espaço em disco.
- Possível necessidade de migração futura para armazenamento em nuvem.
- Uploads são validados por tipo de arquivo via middleware.

(Evidência: backend/middlewares/uploadMiddleware/singleStorageUpload.js, backend/middlewares/uploadMiddleware/uploadMultipleToStorage.js, backend/models/coreModels/Upload.js)
