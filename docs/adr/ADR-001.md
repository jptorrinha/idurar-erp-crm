# Architecture Decision Record 001: Use of Generic CRUD Controller Factory

## Status
Accepted

## Context

The backend requires standard CRUD operations for many entities such as clients, invoices, employees, etc. Writing individual CRUD methods for each model would lead to significant code duplication and maintenance overhead.

## Decision

Implement a generic CRUD controller factory function `createCRUDController` that, given a model name, returns an object with standard CRUD methods (create, read, update, delete, list, search, filter).

## Consequences

- Reduces code duplication and improves maintainability.
- Allows custom controllers to extend or override default CRUD methods as needed.
- Centralizes error handling and response formatting.
- Simplifies adding new entities with standard CRUD operations.
