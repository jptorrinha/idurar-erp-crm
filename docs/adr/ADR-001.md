# ADR-001: Escolha da Stack MERN para o Sistema

## Contexto

O projeto IDURAR ERP CRM foi desenvolvido utilizando a stack MERN (MongoDB, Express.js, React.js, Node.js) para garantir uma aplicação full-stack JavaScript moderna, com alta escalabilidade e flexibilidade. (Evidência: README.md, backend/package.json, frontend/package.json)

## Decisão

Adotar a stack MERN para desenvolvimento do sistema, utilizando:
- MongoDB para banco de dados NoSQL
- Express.js para servidor REST API
- React.js para frontend SPA
- Node.js como runtime backend

## Alternativas Consideradas

- Stack MEAN (Angular em vez de React)
- Backend em outras linguagens (Python, Java)
- Banco de dados relacional (PostgreSQL, MySQL)

## Trade-offs

- MERN permite uso de JavaScript em toda a stack, facilitando desenvolvimento e manutenção.
- MongoDB oferece flexibilidade de esquema, útil para ERP/CRM com dados variados.
- React.js oferece UI reativa e rica, porém requer configuração de bundling (Vite).
- Uso de MongoDB pode dificultar consultas complexas comparado a SQL.

## Consequências

- Equipe deve ter proficiência em JavaScript e tecnologias MERN.
- Dependência de MongoDB exige atenção à modelagem e performance.
- Frontend e backend desacoplados via API REST.
