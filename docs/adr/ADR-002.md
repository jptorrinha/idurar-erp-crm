# Architecture Decision Record 002: JWT Authentication with HTTP-only Cookies

## Status
Accepted

## Context

Secure authentication is critical for the ERP/CRM system. Tokens must be stored securely to prevent XSS attacks.

## Decision

Use JWT tokens for authentication and store them in HTTP-only cookies. Implement middleware to validate tokens on protected routes.

## Consequences

- Tokens are not accessible via JavaScript, reducing XSS risk.
- Simplifies session management with stateless tokens.
- Requires CSRF protections if applicable.
- Allows role-based authorization via token payload.
