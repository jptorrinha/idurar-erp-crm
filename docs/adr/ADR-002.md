# ADR-002: Autenticação via JWT e Middleware de Permissões

## Contexto

O sistema necessita de autenticação segura para acesso às APIs e controle de permissões baseado em roles para diferentes tipos de usuários (admin, staff, etc). (Evidência: backend/routes/coreRoutes/coreAuth.js, backend/middlewares/permission.js)

## Decisão

Implementar autenticação via JSON Web Tokens (JWT) com middleware `isValidAdminToken` para proteger rotas. Utilizar middleware `hasPermission` para verificar permissões específicas baseadas no role do usuário.

## Alternativas Consideradas

- Sessões baseadas em cookies
- OAuth2
- Autenticação via API keys

## Trade-offs

- JWT permite autenticação stateless, escalável para APIs REST.
- Middleware de permissões centraliza controle de acesso.
- JWT pode ter riscos se tokens não forem bem gerenciados (ex: revogação).

## Consequências

- Necessidade de armazenar tokens no cliente com segurança.
- Backend deve validar tokens em todas as rotas protegidas.
- Permissões são definidas em middleware e roles configurados no modelo Admin.

(Evidência: backend/routes/coreRoutes/coreAuth.js, backend/middlewares/permission.js, backend/models/coreModels/Admin.js)
